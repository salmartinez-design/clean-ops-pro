**UPDATED ROW LAYOUT — keep rows at ~60px max**

```
[ Name 14px 500 #1A1917 ]                           [ Freq badge ]
[ Zone dot · Address 12px #4A4845 ]    [ Last done: Apr 2 · Next: Apr 16 — 11px #6B6860 ]
```

---

**ELEMENT 1 — Zone color dot**

Inline on the second line, immediately before the address text. Gap 6px between dot and address.

- Size: 10px × 10px filled circle, border-radius 50%, flex-shrink 0
- Color: match `clients.zip_code` against `service_zones` zip arrays WHERE `company_id = current` — use that zone's `color` hex field
- No zone match: `#B4B2A9` (gray)
- `title` attribute on the dot = zone name for hover tooltip (no custom tooltip needed)

---

**ELEMENT 2 — Frequency badge**

Top-right of the row, one badge only.

- Background `#F0EDE8`, color `#4A4845`, font-size 10px, padding 2px 7px, border-radius 10px, font-weight 500
- Display mapping:

| DB value | Display |
|---|---|
| `weekly` | Weekly |
| `every_2_weeks` / `biweekly` | Biweekly |
| `every_4_weeks` / `monthly` | Monthly |
| `onetime` / `one_time` | One-Time |
| null / missing | No recurring |

Never show raw DB strings.

---

**ELEMENT 3 — Last completed service**

Bottom-right line, left portion.

- Query: MAX(`service_date`) from `job_history` WHERE `client_id = client.id` AND `company_id = current`
- Display: "Last done: [Mon DD]" — e.g. "Last done: Apr 2"
- Format: `MMM D` (no year unless prior year — then show "Mar 28, 2025")
- If no history: "Last done: —"
- Font: 11px, color `#6B6860`

---

**ELEMENT 4 — Last booked and completed**

Bottom-right line, right portion, separated from Element 3 by " · "

- Query: the most recent job from `jobs` WHERE `client_id = client.id` AND `status = 'done'` AND `company_id = current` — return `scheduled_date` and `scope` (or service type)
- If that matches the same date as Element 3 (job_history), that's fine — they're confirming the same event from two angles
- Also query: next upcoming job from `jobs` WHERE `client_id = client.id` AND `scheduled_date >= today` AND `status NOT IN ('void', 'moved', 'skip', 'cancelled')` ORDER BY `scheduled_date` ASC LIMIT 1
- Display format:
  - If upcoming job exists: "Next: [Mon DD]" in `#6B6860`
  - If no upcoming job: "Next: none" in `#A32D2D` (red — signals lapsed client, rep should know)
- Combined bottom-right line: `Last done: Apr 2 · Next: Apr 16`

---

**API RESPONSE — add these fields to the existing search endpoint:**

```json
{
  "zone_color": "#5B9BD5 or null",
  "zone_name": "Oak Lawn Central or null",
  "frequency": "biweekly",
  "last_service_date": "2026-04-02",
  "next_job_date": "2026-04-16 or null"
}
```

Run all supplemental queries (zone, frequency, last service, next job) in `Promise.all` per result — do not run them sequentially. Return max 8 results. Target response under 300ms.
