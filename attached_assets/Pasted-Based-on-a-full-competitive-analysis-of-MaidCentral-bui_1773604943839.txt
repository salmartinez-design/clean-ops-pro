(MC makes this painful, we make it fast)

MaidCentral's job creation is a long form.
Ours should be a guided 3-step wizard.

Replace "+ New Job" button behavior:

STEP 1 — CLIENT (10 seconds):
  Search existing clients — type-ahead search
  Shows client name, address, last service date
  "or + Add New Client" link
  
  If existing client selected:
    Auto-fills address
    Shows last 3 jobs for this client
    Shows any standing notes/preferences
    "Same as last time" quick button

STEP 2 — JOB DETAILS (20 seconds):
  Service Type — large button grid:
    [Standard Clean] [Deep Clean] [Move Out]
    [Recurring] [Post-Construction] [Commercial]
  
  Date — calendar picker
  Time — time selector with common times as pills:
    [8:00 AM] [9:00 AM] [10:00 AM] [Custom]
  
  Duration — auto-suggested based on service type
    and client's home size if on file
    Editable slider: 1h — 2h — 3h — 4h — 5h+
  
  Price — auto-calculated from service type + duration
    Editable field
    Shows "Last charged: $185" if recurring client

STEP 3 — ASSIGN (10 seconds):
  Shows available employees for that date/time
  Each employee card shows:
    Name + photo
    Jobs already scheduled that day
    Distance from job address (if geocoded)
    Current workload: "2 jobs · 4h scheduled"
  
  Click to assign — can assign multiple (teams)
  
  "Schedule Job" button — creates job instantly
  Shows on dispatch board immediately

IMPROVEMENT 3 — AUTOMATED COMMUNICATIONS
(MC has this but it's clunky, ours is elegant)
════════════════════════════════════════════════

DATABASE:
CREATE TABLE IF NOT EXISTS notification_templates (
  id          SERIAL PRIMARY KEY,
  company_id  INTEGER NOT NULL REFERENCES companies(id),
  trigger     VARCHAR(50) NOT NULL,
  channel     VARCHAR(10) NOT NULL 
              CHECK (channel IN ('email','sms')),
  subject     TEXT,
  body        TEXT NOT NULL,
  is_active   BOOLEAN DEFAULT true,
  created_at  TIMESTAMP DEFAULT NOW()
);

CREATE TABLE IF NOT EXISTS notification_log (
  id            SERIAL PRIMARY KEY,
  company_id    INTEGER NOT NULL,
  recipient     TEXT NOT NULL,
  channel       VARCHAR(10),
  trigger       VARCHAR(50),
  status        VARCHAR(10) DEFAULT 'sent',
  sent_at       TIMESTAMP DEFAULT NOW()
);

TRIGGERS (auto-fire these):

CLIENT NOTIFICATIONS:
  24h before job:
    SMS: "Hi [name]! Your [service type] is 
    tomorrow at [time]. 
    Reply SKIP to skip or CHANGE to reschedule.
    - [Company Name]"
  
  When cleaner clocks in:
    SMS: "Your cleaner [first name] has arrived 
    and started your cleaning!"
  
  When job complete:
    SMS: "Your cleaning is complete! 
    Rate your service: [portal link]
    Tip your cleaner: [tip link]"
  
  Invoice sent:
    Email with invoice PDF attached

EMPLOYEE NOTIFICATIONS:
  New job assigned:
    SMS: "New job assigned: [client name] 
    at [address] on [date] at [time]"
  
  Schedule change:
    SMS: "Your schedule was updated. 
    Check your jobs: [my-jobs link]"
  
  Time-off approved:
    SMS: "Your time-off request for [date] 
    has been APPROVED ✓"

NOTIFICATION SETTINGS PAGE 
  Add to Company settings:
  
  Toggle each trigger ON/OFF
  Edit message templates with variables:
    [client_name] [cleaner_name] [date] 
    [time] [address] [company_name] [portal_link]
  
  Test button: "Send Test Message"
  Notification log: last 100 sent messages
