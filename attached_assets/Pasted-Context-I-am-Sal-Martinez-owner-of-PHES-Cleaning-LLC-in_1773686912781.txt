SPRINT 2 — BULK INVOICING + CLOSE DAY

---

PART A — BATCH INVOICE PANEL

Location: /invoices page
Add a "Batch Invoice" button in the top-right header area, 
next to the existing "+ New Invoice" button.
Style: secondary button (white background, brand color border 
and text).
Visible to: owner, admin, office roles only.

When clicked, open a RIGHT SIDE DRAWER panel (not a modal).
The drawer must be full-height, 520px wide on desktop, 
full-width on mobile. It slides in from the right, 
overlaying the page with a dark backdrop.

DRAWER — STEP 1: JOB SELECTION

Drawer header:
- Title: "Batch Invoice"
- Subtitle: "Select completed jobs to invoice"
- X close button top right

Pull all jobs where:
- company_id = current tenant
- status = 'complete'
- No invoice exists yet (no row in invoices table with this 
  job_id, or invoice status = 'draft' only — do not include 
  jobs already invoiced with sent/paid status)

Display as a scrollable list. Each row contains:
- Checkbox (left)
- Client full name (bold)
- Job date (formatted: Mon Mar 15)
- Scope of work / service type
- Assigned technician first name
- Job amount ($XXX.XX) right-aligned
- If amount is $0 or null: show "No amount set" in red 
  secondary text

Above the list:
- "Select All" checkbox that toggles all rows
- Count label: "X jobs ready to invoice" in secondary text
- Search input to filter by client name

Below the list (sticky footer):
- Two toggle rows:
  1. Auto-Send: "Email invoice to client automatically" 
     Toggle off by default. When on, show info text: 
     "Invoice will be sent via email using Resend"
     If RESEND_API_KEY not configured: toggle disabled, 
     tooltip "Resend not configured"
  2. Auto-Charge: "Charge card on file automatically"
     Toggle off by default. When on, show warning in amber:
     "Client must have a saved payment method. Failed 
     charges will be skipped and flagged."
     If STRIPE_SECRET_KEY not configured: toggle disabled,
     tooltip "Stripe not configured"
- Running total: "X jobs selected — Total: $X,XXX.XX"
  Updated in real time as checkboxes change
- Primary button: "Generate Invoices" (brand color, full width)
  Disabled if 0 jobs selected
- Secondary link: "Cancel"

Validation before processing:
- If Auto-Charge is ON and any selected job's client has no 
  payment method on file: show a warning step before 
  processing. List the clients with no payment method and ask:
  "Skip these clients or cancel?"
  Two buttons: "Skip and Continue" | "Cancel"

DRAWER — STEP 2: PROCESSING

Replace the job list with a processing view:
- Progress bar (animated, brand color fill)
- Current item text: "Invoicing [Client Name]..."
  Updates as each invoice is processed
- Count: "X of Y complete"
- Do NOT close the drawer during processing
- Process invoices sequentially, not in parallel

Processing logic per job:
1. Create invoice record in invoices table:
   - invoice_number: auto-generated (INV-[YEAR]-[4-digit seq])
   - company_id, job_id, client_id from job record
   - amount: from job.amount
   - status: 'draft' initially
   - due_date: today + 7 days
   - line_items JSONB: [{description: scope name, 
     amount: job.amount}]
   - created_by: current user id

2. If Auto-Send is ON and RESEND_API_KEY configured:
   - Generate invoice PDF
   - Send via Resend to client email
   - Update invoice status to 'sent'
   - Update invoice.sent_at to now()
   - Log to notification_log

3. If Auto-Charge is ON and STRIPE_SECRET_KEY configured 
   and client has payment method:
   - Attempt Stripe charge for invoice amount
   - On success: update invoice to 'paid', create payment 
     record, update job.paid = true
   - On failure: mark invoice back to 'sent', flag 
     payment_failed = true, log the error

DRAWER — STEP 3: SUMMARY

After all processing complete, show summary screen:
- Use an SVG checkmark icon (no emoji)
- Heading: "Batch Complete"
- Four stat cards in a 2x2 grid:
  - Invoices Created: X (green)
  - Emails Sent: X (blue) — only if Auto-Send was ON
  - Payments Collected: $X,XXX.XX (green) — only if 
    Auto-Charge was ON
  - Failed / Skipped: X (red) — only if any failures
- If any failures: expandable list of failed clients 
  with reason
- Button: "Close" — closes drawer, refreshes invoice list
- Button: "View Invoices" — closes drawer, stays on 
  /invoices filtered to today's invoices

PART B — CLOSE DAY WORKFLOW

Location: Add "Close Day" button to TWO places:
1. Dashboard page — top-right header area
2. /invoices page — top-right header next to "Batch Invoice"

Style: outlined secondary button.
Visible to: owner and admin roles only.

When clicked, open a CENTERED MODAL.
Modal: 640px wide, auto height, soft shadow.

MODAL — CLOSE DAY REVIEW

Header:
- Title: "Close Day — [Day, Month Date]"
- Subtitle: "Review today's activity before closing"
- X close button

Four checklist sections. Each has a status icon:
green checkmark if clear, amber warning if attention needed, 
red alert if action required.

SECTION 1 — JOBS TODAY
Pull all jobs where job_date = today:
- Complete: X jobs (green if > 0)
- In Progress: X jobs (amber if > 0)
- Scheduled not started: X jobs (red if after 5pm, 
  amber if before)
- Flagged: X jobs (red if > 0, link to Clock Monitor)

SECTION 2 — INVOICING
- Invoiced: X of Y completed jobs today
- If X < Y: amber warning, show count of uninvoiced jobs
- Button: "Batch Invoice Now" — opens Batch Invoice drawer
- If all invoiced: green checkmark

SECTION 3 — PAYMENTS TODAY
- Total collected: $X,XXX.XX (bold, brand color)
- Invoices sent awaiting payment: X (blue)
- Overdue invoices: X, $X,XXX.XX (red if > 0)
- If any overdue: "Send reminders" link — bulk sends 
  reminder emails to all overdue clients

SECTION 4 — CLOCK ENTRIES
- Clocked in and out: X entries (green)
- Missing clock-out: X (red if > 0)
  Show each employee name with missing clock-out
  Inline button: "Set Clock Out" — time picker to manually 
  enter clock-out time
- Flagged entries: X (red if > 0, link to Clock Monitor)

MODAL FOOTER:
- Left: "Today's Revenue" — sum of all paid invoices today
  Large number, brand color
- Right two buttons:
  1. "Close" — closes modal, no action
  2. "Mark Day Complete" — saves daily_summary record,
     closes modal, shows toast: "Day marked complete."
     Disabled with tooltip "Uninvoiced jobs remaining" 
     if Section 2 shows uninvoiced jobs

Database — add this table:
daily_summaries:
  id, company_id, summary_date (date, unique per company),
  jobs_complete (int), jobs_flagged (int),
  invoices_created (int), invoices_sent (int),
  revenue_collected (numeric), revenue_outstanding (numeric),
  clock_entries_missing (int), 
  marked_complete_by (uuid → users),
  marked_complete_at (timestamptz), notes (text)
RLS: company_id must match current tenant.

PART C — INVOICE PAGE + DETAIL IMPROVEMENTS

1. INVOICE LIST PAGE — add columns:
   - "Due Date" column (after Amount)
   - "Days Overdue" — red badge "X days overdue" if 
     status = sent and due_date < today

2. INVOICE DETAIL PAGE — add action buttons:

   "Charge Now":
   - Visible when: status = sent or overdue, client has 
     payment method, STRIPE_SECRET_KEY configured

   - Visible when: status = sent or overdue

     Other), Amount (pre-filled), Date (default today), 
     Notes (optional)
   - On save: creates payment record, updates to paid

3. ADD OVERDUE STATUS:
   - Any invoice where status = sent AND due_date < today 
     displays as OVERDUE (red badge)
   - Check on page load — no cron needed yet
   - Add Overdue to filter tabs on /invoices

4. UPDATE SUMMARY CARDS on /invoices:
   - Outstanding: sum of all sent invoices not yet paid
   - Overdue: sum of sent invoices where due_date < today
   - Paid (30 days): sum of payments in last 30 days
   - YTD: sum of all payments this calendar year
   All amounts: $X,XXX format (no decimals on cards)

status ('sent'/'failed'), metadata JSONB, sent_at.
