REPLIT PROMPT — Quote Builder: Returning Client Quick Book + UX Fixes
Targeted enhancements to Step 1 of /quotes/new when an existing client is loaded. No changes to any other step or page.

1. HIDE "HOW DID YOU HEAR ABOUT US" FOR EXISTING CLIENTS
When a client is selected from the existing client search (client record loaded into the form), hide the "How did you hear about us?" field entirely — do not just disable it, remove it from the DOM conditionally.
Set the field value silently to "existing_client" in the form state so the API payload remains valid.
Only show the field when the user clicked "Enter lead info instead" (new lead flow).

2. PREFERRED TECHNICIAN SUGGESTION
When an existing client is loaded, query the job_history table for that client: find the technician who appears most frequently across their last 20 jobs (assigned_technician_id, COUNT descending, LIMIT 1).
If a preferred tech is found, show a suggestion pill directly below the zone banner:
Background: #EBF4FF
Border: 1px solid #5B9BD5
Border-radius: 6px
Padding: 8px 12px
Font-size: 12px
Color: #185FA5
Text: "Preferred tech: [Full Name] — assigned to most of this client's jobs"
Add a subtle "Use this tech" button inline (same line, right side) — ghost style, 11px, brand color. On click: pre-selects this technician in the Phase 2 tech suggestion list (carries forward to job assignment on save).
If no clear preferred tech (all jobs unassigned or no history): show nothing.

3. QUICK BOOK PANEL
When an existing client is loaded, render a "Quick Book" panel directly below the zone/tech suggestion area, above the "Next: Property Details →" button.
Panel style:
Background: #F7F6F3
Border: 1px solid #E5E2DC
Border-radius: 8px
Padding: 14px 16px
Panel label: "Quick Book" in 12px font-weight 500 #1A1917, subtext: "Book based on a previous service" in 11px #6B6860
Logic for which options to show:
Query job_history WHERE client_id = selected_client.id AND company_id = current ORDER BY service_date DESC LIMIT 20.
From those results, extract distinct service types (scope field). Rules:

Take up to 3 most recent jobs
If multiple jobs share the same scope/service type, deduplicate — show that scope only once regardless of how many times it appears
Result: 1–3 unique service type options max
If all 20 most recent jobs are the same scope: show only 1 option

Each quick book option renders as a clickable card in a horizontal row (flex, gap 10px):
Card: background #FFFFFF, border 0.5px solid #E5E2DC, border-radius 8px
Padding: 10px 14px
Min-width: 180px
On hover: border-color #5B9BD5, background #EBF4FF
Cursor: pointer
Card contents:

Service type name: 13px, font-weight 500, #1A1917 (e.g. "Deep Clean", "Standard Clean")
Last booked date: 11px, #6B6860 (e.g. "Last: Mar 28")
Price from last booking: 13px, font-weight 500, #1A1917 (e.g. "$220")
Frequency badge if recurring: 10px pill, background #F0EDE8, color #4A4845

On click of a quick book card:

Skip Steps 2 and 3 entirely — pre-populate the quote with:

Scope = that service type
Price = last booked price for that scope
Frequency = that client's current recurring frequency (or one-time if no recurring)
Assigned tech = preferred tech if identified in step 2 above
Add-ons = same add-ons from that last job (pull from job_history add-ons field if stored, otherwise skip)


Navigate directly to Step 4 (Review) with all fields pre-filled
Show a dismissible banner at the top of Step 4: "Pre-filled from [Service Type] on [Date]. Adjust anything before saving." in the blue info style (background: #EBF4FF, border: 1px solid #5B9BD5, color: #185FA5, 12px)

If the user wants to build a custom quote instead, show a text link below the quick book cards: "Build custom quote instead →" in 12px #5B9BD5 — clicking this hides the quick book panel and proceeds to Step 2 normally.
