## DATABASE SCHEMA

Create these tables in Supabase with RLS scoped to company_id:

**scopes_of_work**
- id (uuid, PK)
- company_id (uuid, FK)
- name (text) — e.g. "Deep Clean or Move In/Out", "Standard Clean"
- scope_group (text) — "Residential" or "Commercial"
- hourly_rate (numeric) — base hourly rate for this scope
- minimum_bill (numeric) — minimum charge regardless of hours
- is_active (boolean, default true)
- sort_order (integer)
- created_at, updated_at

**scope_tiers**
- id (uuid, PK)
- scope_id (uuid, FK → scopes_of_work)
- company_id (uuid, FK)
- min_sqft (integer)
- max_sqft (integer)
- hours (numeric) — estimated hours for this sqft range
- created_at

**scope_frequency_factors**
- id (uuid, PK)
- scope_id (uuid, FK → scopes_of_work)
- company_id (uuid, FK)
- frequency (text) — "weekly", "biweekly", "monthly", "onetime"
- rate_override (numeric, nullable) — if set, overrides the scope hourly_rate for this frequency
- multiplier (numeric, default 1.0) — applied to base price if no rate_override
- label (text) — display label e.g. "Weekly", "Bi-Weekly", "Monthly", "One Time"
- sort_order (integer)

**scope_addons**
- id (uuid, PK)
- scope_id (uuid, FK → scopes_of_work)
- company_id (uuid, FK)
- name (text)
- price (numeric, nullable) — flat price
- price_type (text) — "flat" or "percent"
- percent_of_base (numeric, nullable) — used if price_type = "percent"
- time_add_minutes (integer, default 0)
- unit (text, default "each")
- is_active (boolean, default true)
- sort_order (integer)

**scope_discount_codes**
- id (uuid, PK)
- company_id (uuid, FK)
- code (text)
- description (text)
- discount_type (text) — "flat" or "percent"
- discount_value (numeric)
- is_active (boolean, default true)
- created_at

**scope_fee_rules**
- id (uuid, PK)
- company_id (uuid, FK)
- rule_type (text) — "skip_fee", "lockout_fee", or "custom"
- label (text)
- charge_percent (numeric) — percentage of job total to charge (e.g. 100 for 100%)
- tech_split_percent (numeric) — percentage of the fee that goes to the tech (e.g. 40)
- window_hours (integer, nullable) — hours before job that trigger this rule (e.g. 48)
- is_active (boolean, default true)

Apply RLS policies on all tables: authenticated users can only read/write rows where company_id matches their JWT company_id.

## API ROUTES

Create these routes in a new file: routes/pricing.ts

GET    /api/pricing/scopes                    — list all scopes for company
POST   /api/pricing/scopes                    — create scope
PUT    /api/pricing/scopes/:id                — update scope
DELETE /api/pricing/scopes/:id                — soft delete (set is_active = false)

GET    /api/pricing/scopes/:id/tiers          — list tiers for a scope
POST   /api/pricing/scopes/:id/tiers          — bulk replace tiers (accept array, delete existing, insert new)

GET    /api/pricing/scopes/:id/frequencies    — list frequency factors for a scope
POST   /api/pricing/scopes/:id/frequencies    — bulk replace frequency factors

GET    /api/pricing/scopes/:id/addons         — list add-ons for a scope
POST   /api/pricing/scopes/:id/addons         — create add-on
PUT    /api/pricing/addons/:id                — update add-on
DELETE /api/pricing/addons/:id                — soft delete add-on

GET    /api/pricing/discounts                 — list discount codes for company
POST   /api/pricing/discounts                 — create discount code
PUT    /api/pricing/discounts/:id             — update discount code
DELETE /api/pricing/discounts/:id             — delete discount code

GET    /api/pricing/fees                      — list fee rules for company
POST   /api/pricing/fees                      — create fee rule
PUT    /api/pricing/fees/:id                  — update fee rule
DELETE /api/pricing/fees/:id                  — delete fee rule

POST   /api/pricing/calculate                 — calculate price for a given scope, sqft, frequency, and selected add-on ids. Returns: base_hours, hourly_rate, base_price, addons_total, subtotal, minimum_applied (boolean). This is the core calculation endpoint used by the quote wizard and the iFrame booking.

All routes require authentication. All queries filter by req.auth.companyId.

## PRICING CALCULATION LOGIC

The POST /api/pricing/calculate endpoint accepts:
- scope_id
- sqft
- frequency
- addon_ids (array)
- discount_code (optional string)

Calculation steps:
1. Find the tier row where sqft falls between min_sqft and max_sqft
2. Get hours from that tier
3. Get the hourly rate: if frequency factor has a rate_override, use that. Otherwise use scope.hourly_rate × frequency factor.multiplier
4. base_price = hours × hourly_rate
5. If base_price < scope.minimum_bill, use minimum_bill and set minimum_applied = true
6. Calculate each add-on: flat price or (percent_of_base / 100) × base_price
7. addons_total = sum of all selected add-on prices
8. subtotal = base_price + addons_total
9. If discount_code provided: find active matching code. Apply flat or percent discount to subtotal. Return discount_amount and final_total.
10. Return all values so the UI can display a full line-item breakdown.

## COMPANY SETTINGS UI

Add a new "Pricing & Scopes" tab to the Company Settings page. It should appear after the Branding tab in the tab order.

The tab has four sections accessible via sub-tabs or accordion:

**Section 1 — Scopes of Work**
- List of all scopes with name, group (Residential/Commercial), hourly rate, minimum bill, active toggle
- Add New Scope button → inline form or modal: name, scope group, hourly rate, minimum bill
- Each scope row is expandable or has an Edit button
- Drag to reorder scopes (updates sort_order)

**Section 2 — Pricing Tiers** (shown when a scope is selected/expanded)
- Table with columns: Min Sqft, Max Sqft, Hours
- Inline editable rows — click a cell to edit
- Add Row button appends a new row
- Delete row button on each row
- Save Tiers button sends bulk replace to POST /api/pricing/scopes/:id/tiers
- Tiers must be contiguous — validate that max_sqft of row N equals min_sqft of row N+1

**Section 3 — Frequency Factors** (shown when a scope is selected/expanded)
- Table with columns: Frequency, Label, Rate Override, Multiplier
- One row per frequency: weekly, biweekly, monthly, onetime
- Inline editable
- Save button

**Section 4 — Add-Ons** (shown when a scope is selected/expanded)
- Cards or table rows for each add-on: name, price type (flat/percent), price/percent value, time add, active toggle
- Add Add-On button
- Delete button per add-on

**Section 5 — Discount Codes** (global, not per-scope)
- Table: Code, Description, Type, Value, Active toggle
- Add Discount Code button → inline form
- Delete button per code

**Section 6 — Fee Rules** (global, not per-scope)
- Table: Rule Type, Label, Charge %, Tech Split %, Window Hours, Active toggle
- Add Fee Rule button
- Delete button per row
