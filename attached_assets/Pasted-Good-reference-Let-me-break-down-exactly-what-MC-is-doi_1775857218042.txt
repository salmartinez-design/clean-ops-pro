Good reference. Let me break down exactly what MC is doing across these 6 screenshots and map it against where Qleno stands:

**What MC does that Qleno already matches or beats:**
- Scope selection with add-ons per scope — built
- Discount codes — built (Manager50, Compass, Law Enforcement, Senior, Realtor)
- Client info persistent on the side — built (right panel)
- Price preview — built
- Job notes / special instructions — built

**What MC does that Qleno is missing:**

The biggest one is the **left sidebar tab navigation** (Image 6 — Profile / Pricing / Booking / Scheduling). MC lets the rep jump non-linearly between sections without losing state. Your stepper forces linear progression. For a phone call workflow that's a problem — the client might give you the date before they tell you sqft.

**+/- quantity counters on add-ons** (Images 2–5) — MC allows adding the same add-on multiple times (e.g. 2 ovens). Qleno uses checkboxes. For most residential jobs this doesn't matter but for Move-Out or commercial it does.

**"One Time" vs recurring toggle per add-on** — each add-on in MC can be marked one-time or recurring independently. Qleno doesn't have this.

**Internal Memo separate from Special Instructions** (Image 6) — two distinct text fields. Special Instructions go to the client/tech. Internal Memo stays office-only. Qleno has Job Notes (tech-visible) and Client-Facing Notes but no pure internal memo.

**Adjustment(+) and Adjustment(-) line items** (Images 3–4) — free-form dollar amount overrides. Lets the rep add or subtract arbitrary amounts with a reason. Essential for edge cases.

**"How many hours would you like to book?"** field — for hourly scopes specifically, MC lets the office manually set hours. Qleno calculates from sqft automatically which is better, but you need a manual override for edge cases.

**What MC does that we should NOT copy:**
- The TinyMCE rich text editor for notes — massive overkill, plain textarea is correct
- The +/- counter UI for on/off add-ons — confusing for simple yes/no items. Only use counters where quantity actually matters (oven count, etc.)
- Everything on one infinite scroll page — your wizard is cleaner
- "Rate Modifications" as a label — nobody knows what that means
- Two-column Cleaning Extras / Other split — arbitrary and confusing
- "Company Zone" in magenta — your zone color dot approach is better

**1. NON-LINEAR STEP NAVIGATION**

**2. ADD-ON QUANTITY COUNTERS (where applicable)**

In Step 3 (Add-ons), change the following add-ons from checkbox to a +/- quantity counter:

- Oven Cleaning (a home can have 2 ovens)
- Refrigerator Cleaning (a home can have 2 fridges)
- Kitchen Cabinets (quantity = number of cabinet sets)

All other add-ons (Windows, Clean Basement, Parking Fee) remain as checkboxes — they are on/off only.

Counter UI: `[ − ] [ 0 ] [ + ]` inline, same row as the add-on name and price. Min value 0, max value 5. When value > 0, price multiplies accordingly (e.g. 2 ovens = $100).

**3. ONE-TIME vs RECURRING TOGGLE PER ADD-ON**

For each add-on in Step 3, add a small toggle to the right of the price: `[ One Time ] [ Recurring ]` — two-button toggle, default "One Time."

When "Recurring" is selected: the add-on applies to every future job in the recurring schedule (stored as `recurring: true` on the add-on record in the quote payload).

Style: 10px text, background `#F0EDE8`, selected state background `#5B9BD5` white text, border-radius 4px, padding 2px 8px.

**4. MANUAL PRICE ADJUSTMENT LINE ITEMS**

In Step 3 (Add-ons), below all add-ons and above the scope subtotal, add two optional line items:

**Adjustment (+):**
- Label: "Manual adjustment (+)" in 13px `#1A1917`
- Input: dollar amount field, placeholder "$0.00", right-aligned
- Reason field: text input, placeholder "Reason (e.g. extra floor, large home)", 12px
- When amount > 0: adds to scope subtotal, shows as a line item in price preview

**Adjustment (−):**
- Same as above but subtracts from subtotal
- Reason field same
- Validate: cannot make subtotal go below $0

Both adjustments stored on the quote payload as `manual_adjustments: [{type: 'add'|'subtract', amount: X, reason: '...'}]`. Add column if needed: `ALTER TABLE quotes ADD COLUMN IF NOT EXISTS manual_adjustments JSONB`.
