Rebuild the Reschedule Job modal into a full rescheduling 
workflow. The current modal is too bare — it only shows 
a date and time picker with no context. Replace it with 
a multi-section modal that gives the owner everything 
needed to make a rescheduling decision without leaving 
the screen.

DESIGN SYSTEM — NON-NEGOTIABLE:
- Background: #F7F6F3
- Cards: #FFFFFF
- Primary text: #1A1917
- Secondary text: #6B6860
- Borders: #E5E2DC
- Accent: var(--brand) per tenant
- Font: Plus Jakarta Sans only
- No emojis anywhere
- No dark mode
- Light theme only

---

THE NEW RESCHEDULE MODAL

The modal should have four sections stacked vertically.
Keep it scrollable. No tabs — everything visible 
in one flow.

---

SECTION 1 — JOB SUMMARY (read only, compact)

Show at the top so context is always visible:
- Client name
- Service type
- Current date and time
- Current assigned tech(s)
- Job price

Example: "Amanda Thornton — Deep Clean — 
Mar 22 at 8:30 AM — Ana Valdez — $195.00"

---

SECTION 2 — REASON FOR RESCHEDULE (required)

A required reason selector before anything else.
Options:
- Client Request
- No Show — Client
- No Show — Tech
- Weather
- Tech Unavailable
- Emergency
- Other (shows a text field)

This is required. Confirm Reschedule button is 
disabled until a reason is selected.
Reason is logged to cancellationLogTable with 
the rescheduled job ID.

---

SECTION 3 — NEW DATE + AVAILABILITY VIEW

Date picker (same as current).

After a date is selected, immediately below show 
an availability grid for that date — do not make 
the user go check the dispatch board separately.

The availability grid shows:
- Each time slot from 7 AM to 5 PM in 1-hour rows
- For each slot: how many jobs are already 
  scheduled at that time
- Color coding:
  - Open (0 jobs): light green background
  - Busy (1-2 jobs): amber background  
  - Full (3+ jobs): light red background
- The currently selected time slot is highlighted 
  with the brand accent color
- Tapping a time slot selects it as the new time

This replaces the basic time text field with a 
visual availability picker. The owner can see 
at a glance which slots are open without switching 
to the dispatch board.

Load the availability data via 
GET /api/jobs/availability?date=YYYY-MM-DD
which returns job counts per hour slot for 
that date scoped to the company.

---

SECTION 4 — TEAM ASSIGNMENT

Show the currently assigned tech(s) for this job.

Below that, show available techs for the 
selected new date and time:
- Pull from employee availability 
  (GET /api/employees/available?date=X&time=Y)
- Show each available tech as a selectable card:
  Name, jobs that day, zone
- Currently assigned tech shown first with 
  a "Currently assigned" label
- If the currently assigned tech has a conflict 
  at the new time, show a red warning: 
  "Ana Valdez has a conflict at this time"
- Owner can keep the same tech or tap a 
  different tech to reassign
- Multi-tech jobs: show all current techs, 
  allow swapping any of them

---

CONFIRM BUTTON

At the bottom, a full-width "Confirm Reschedule" 
button. Disabled until:
- Reason is selected
- New date is selected
- New time is selected
- At least one tech is assigned

On confirm:
- Updates job scheduled_date, scheduled_time, 
  assigned_employee_id
- Logs to cancellationLogTable: 
  job_id, reason, rescheduled_to_date, 
  rescheduled_to_time, rescheduled_by, 
  original_date, original_time
- If Resend is configured: sends a reschedule 
  notification email to the client with the 
  new date/time (if RESEND_API_KEY exists)
- Shows success: "Job rescheduled to 
  [new date] at [new time] with [tech name]"
- Closes modal and refreshes dispatch board

---

MOBILE REQUIREMENTS

This modal must work on mobile (375px viewport):
- Full screen bottom sheet on mobile 
  (slides up from bottom, scrollable)
- Availability grid time slots must be 
  tap-friendly (minimum 44px height per row)
- Tech cards must be full width on mobile
- Confirm button fixed at bottom of sheet 
  so it is always reachable
