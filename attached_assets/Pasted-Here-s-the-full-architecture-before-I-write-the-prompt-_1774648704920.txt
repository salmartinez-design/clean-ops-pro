Here's the full architecture before I write the prompt so you can see exactly what's being built:

**What the engine does:**
- Reads all active `recurring_schedules` (86 imported + any new widget bookings)
- For each schedule, calculates the next N job dates based on frequency
- Generates job records in the `jobs` table with correct date, time, technician(s), scope, address
- Runs on a rolling 8-week window — always keeps 8 weeks of jobs ahead
- Runs automatically every night at midnight + on demand via an admin trigger
- Never duplicates — checks before inserting

**Edge cases handled:**
- 9 schedules with no technician → generate as Unassigned
- Paused/cancelled schedules → skip
- First run seeds all jobs from today forward through 8 weeks
- New widget recurring bookings → picked up on next engine run automatically

**PART 2 — FREQUENCY CALCULATION LOGIC**

Create a utility function `getNextOccurrences(schedule, fromDate, weeksAhead)` in `server/utils/recurringEngine.ts`:

Frequency mappings (match values stored in recurring_schedules.frequency):
- `'weekly'` → add 7 days per occurrence
- `'biweekly'` or `'every_2_weeks'` → add 14 days
- `'every_3_weeks'` → add 21 days
- `'every_4_weeks'` or `'monthly'` → add 28 days

Logic:
1. Start from `fromDate` (today if no jobs exist, or `last_generated_through + 1` if already partially generated)
2. Find the first occurrence on or after `fromDate` that falls on the correct day of week (use `schedule.day_of_week` — 0=Sunday through 6=Saturday)
3. Generate all occurrences from that first date through `fromDate + weeksAhead * 7`
4. Return array of dates

If `schedule.day_of_week` is null, use the day of week from `schedule.start_date` or the first job's date as the anchor.

**PART 3 — JOB GENERATION FUNCTION**

Create `generateJobsForSchedule(schedule, weeksAhead = 8)` in the same file:

```
For each date returned by getNextOccurrences:
  1. Check if a job already exists for this schedule_id + date (skip if yes)
  2. Build job record:
     - company_id: schedule.company_id
     - client_id: schedule.client_id
     - recurring_schedule_id: schedule.id
     - scheduled_date: calculated date
     - scheduled_time: schedule.scheduled_time (e.g. "09:00")
     - scope: schedule.scope
     - service_address: schedule.service_address
     - status: 'scheduled'
     - frequency: schedule.frequency
     - source: 'recurring_engine'
  3. INSERT INTO jobs with ON CONFLICT DO NOTHING
  4. After insert, pull the new job's id
  5. Pull technicians from recurring_schedule_technicians WHERE schedule_id = schedule.id
  6. INSERT into job_technicians for each technician
     - If no technicians found: leave job_technicians empty (job shows as Unassigned on board)
  
After all dates processed:
  UPDATE recurring_schedules SET last_generated_through = [furthest date generated] WHERE id = schedule.id
```

**PART 4 — MAIN ENGINE RUNNER**

Create `runRecurringEngine(companyId = null)` in the same file:

```
1. Query recurring_schedules WHERE status = 'active'
   - If companyId provided: filter to that company only
   - If null: run for ALL companies (for the nightly cron)

2. For each schedule:
   - Call generateJobsForSchedule(schedule, 8)
   - Log: "Generated X jobs for schedule [id] — client [client_id]"
   - On error: log the error and continue to next schedule (never abort the full run)

3. Return summary: { schedulesProcessed, jobsCreated, errors }
```

import cron from 'node-cron';

// Run every night at midnight Central time
