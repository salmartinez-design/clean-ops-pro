PART 2 — JOB STATUS SMS NOTIFICATIONS
════════════════════════════════════════════════

DATABASE:
ALTER TABLE jobs ADD COLUMN IF NOT EXISTS
  status_on_my_way_at TIMESTAMP,
  status_started_at TIMESTAMP,
  status_paused_at TIMESTAMP,
  status_finished_at TIMESTAMP,
  rating_requested_at TIMESTAMP,
  client_rating INTEGER CHECK (client_rating BETWEEN 1 AND 4),
  client_rating_comment TEXT,
  client_rating_at TIMESTAMP;

CREATE TABLE IF NOT EXISTS job_status_log (
  id          SERIAL PRIMARY KEY,
  company_id  INTEGER NOT NULL,
  job_id      INTEGER NOT NULL REFERENCES jobs(id),
  user_id     INTEGER NOT NULL REFERENCES users(id),
  status      VARCHAR(20) NOT NULL,
  lat         DECIMAL(10,8),
  lng         DECIMAL(11,8),
  created_at  TIMESTAMP DEFAULT NOW()
);

API ROUTES:

POST /api/jobs/:jobId/status
  Body: { status, lat, lng }
  Status options: 'on_my_way' | 'started' | 
                  'paused' | 'finished'
  
  1. Validate: user is assigned to this job
  2. Update job status timestamp
  3. Insert into job_status_log
  4. Send SMS to client via Twilio
  5. Return { success: true, status, timestamp }

POST /api/jobs/:jobId/rate
  Body: { rating, comment }
  Called when client replies to SMS
  Or submits via portal link
  
  1. Save rating + comment to jobs table
  2. Create scorecard_entry record
  3. Return { success: true }

SMS MESSAGES (use company notification templates):

ON MY WAY:
  "Hi [client_first_name]! [cleaner_first_name] 
   is on the way to your home and will arrive 
   in approximately 15 minutes. 🧹
   — [company_name]"

STARTED (auto-fires on clock-in):
  "Your cleaning has started! 
   [cleaner_first_name] is at your home now. 
   We'll let you know when we're done.
   — [company_name]"

PAUSED:
  "Your cleaner stepped out briefly 
   (supply run or short break) and will 
   be back shortly to finish up!
   — [company_name]"

FINISHED (auto-fires on clock-out):
  "Your home is fresh and clean! ✨
   How did we do today? Reply with:
   4 - Excellent 😊
   3 - Good 👍  
   2 - Fair 😐
   1 - Poor 😞
   — [company_name]"

RATING FOLLOW-UP (fires 30 min after FINISHED 
if no rating received):
  "Hi [client_first_name], 
   it only takes 5 seconds — 
   how was today's cleaning?
   Reply 4 (Excellent), 3 (Good), 
   2 (Fair), or 1 (Poor)
   — [company_name]"

TWILIO INBOUND WEBHOOK:
  POST /api/webhooks/sms-reply
  When client replies with 1/2/3/4:
    1. Find the most recent finished job 
       for that phone number
    2. Save rating to jobs table
    3. Create scorecard_entry
    4. Send confirmation SMS:
       If 4 or 3: 
         "Thank you! ⭐ We're so glad you're 
          happy. See you next time!
          — [company_name]"
       If 2 or 1:
         "Thank you for letting us know. 
          A manager will follow up with you 
          shortly to make things right.
          — [company_name]"
         AND: Create a contact_ticket 
              automatically with type 'complaint'
              AND: Send internal alert to 
              company owner/admin
